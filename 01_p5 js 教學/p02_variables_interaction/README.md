# p02 變數與互動 (variables & interaction)

教學步驟 2。延續上一步的基本繪圖，這一步開始讓畫面「動起來」並且「能被操作」。

## 這一步學什麼

- 用變數 (variable) 儲存會變化的狀態，讓畫面不再是靜止的。
- 理解 p5.js 的動畫迴圈：`setup()` 與 `draw()` 各自負責什麼。
- 加入互動 (interaction)：圖形跟隨滑鼠，並用滑鼠 / 鍵盤事件改變畫面。

## 核心概念

### 1. 變數 (variable)

`sketch.js` 最上方宣告了幾個全域變數（如 `radius`、`hue`、`trail`、`dots`），
用來記住目前的狀態。因為宣告在最外層，`setup()` 和 `draw()` 都能讀寫它們。
動畫的本質，就是「每一幀稍微改變這些變數，再重畫一次」。

### 2. setup() 與 draw() 動畫迴圈 (animation loop)

- `setup()`：程式啟動時只執行「一次」，用來做一次性設定（建立畫布、設定色彩模式等）。
- `draw()`：由 p5.js 自動「重複呼叫」，預設約每秒 60 次（60 fps，frames per second）。
  每呼叫一次就畫出「一幀 (frame)」。每幀先用 `background()` 清畫面，再依目前變數重畫，
  連續播放就形成動畫。本範例用 `sin()` 讓半徑來回脹縮，做出「呼吸 (breathing)」效果。

### 3. 互動事件函式 (event functions)

- `mouseX` / `mouseY`：p5.js 隨時更新的內建變數，代表滑鼠在畫布上的座標，
  本範例用它讓主圓跟隨滑鼠並畫出尾巴軌跡。
- `mousePressed()`：滑鼠按下時，p5.js 會自動呼叫一次，這裡用來新增一個固定圓點。
- `keyPressed()`：鍵盤按下時自動呼叫一次，這裡用來換色或清空圓點。

這三者都不是我們主動去呼叫的，而是「事件發生時 p5.js 幫我們呼叫」，這就是事件驅動 (event-driven)。

## 如何執行

不需要任何安裝或伺服器：用瀏覽器直接打開本資料夾中的 `index.html` 即可
（`index.html` 會從 CDN 載入 p5.js，再載入 `sketch.js`）。

## 操作說明

- 移動滑鼠：主圓與淡出的軌跡尾巴會跟隨滑鼠移動。
- 按下滑鼠（點擊）：在點擊位置留下一個隨機顏色的固定圓點。
- 按任意鍵：隨機改變主圓的色相 (hue)。
- 按 `c` 鍵：清空所有點擊產生的圓點。

畫面左上角會即時顯示目前的 fps、hue 與圓點數量，方便對照變數的變化。

## 可調參數

於 `sketch.js` 中可自行調整：

- `createCanvas(640, 480)`：畫布大小。
- `radius = 60`：主圓基準半徑；`sin(breathPhase) * 20` 中的 `20` 是呼吸脹縮幅度。
- `breathPhase += 0.05`：呼吸速度，數值越大脹縮越快。
- `trail.length > 25` 的 `25`：軌跡尾巴保留的點數（越多尾巴越長）。
- `colorMode(HSB, 360, 100, 100)`：色彩模式與範圍。
- `circle(d.x, d.y, 30)` 的 `30`：點擊圓點的大小。
