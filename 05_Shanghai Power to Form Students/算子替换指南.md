# 算子替换指南 —— 复制粘贴,换一个你自己的算子

进阶册的核心承诺:**权力体制 = 算子的配方,而算子可教、可改、可复制粘贴。**
这份说明让你不必读懂 `engine/`,就能加 / 换一个算子。三步:**写 → 登记 → 用进配方**。

---

## 0. 一个算子长什么样

算子就是一个**纯函数**:吃一批楼、吐一批楼。

```python
def my_operator(recs, target, ...):
    return new_recs
```

- `recs` = 楼的列表,每栋是一个 dict:
  | 键 | 意思 |
  |---|---|
  | `geom` | footprint 几何(shapely Polygon/MultiPolygon,单位**米**,EPSG:32651) |
  | `h` | 高度(米) |
  | `sh` | 权利方:`state` / `developer` / `resident` / `unknown` |
  | `frozen` | 是否被 `freeze` 锁定(`True` 就别动它) |
- **不要改原始 `recs`**:先 `out = [dict(r) for r in recs]`,改 `out`,返回 `out`。
- **GFA 约定**:某栋 GFA ∝ `geom.area × h`。想「守恒」就缩面积时把高度补回来(`h /= ratio`);想「加量」就只抬 `h`。
- `target` 惯例:一个权利方名单(如 `["resident","developer"]`),只作用在名单内、且没被 frozen 的楼。
  判断用:`r["sh"] in target and not r.get("frozen")`。

## 1. 写:复制这个模板,改中间三行

把 `engine/my_operator.py` 打开(已附一个可跑的例子),或直接在 notebook 里粘这段:

```python
import shapely.affinity as aff
import common as C   # 需要 C._polys / C._scale 之类工具时用

def my_op(recs, target, amount=1.5):
    """一句话说明:这个算子对目标楼做了什么形态操作。"""
    out = [dict(r) for r in recs]          # ① 复制,别改原始
    for r in out:
        if r["sh"] in target and not r.get("frozen"):   # ② 只动目标、未冻结
            # ③ —— 在这里写你的形态操作。下面三个是常见动作,挑一个改：——
            r["h"] = r["h"] * amount                              # a. 抬高度(加 GFA)
            # r["geom"] = aff.rotate(r["geom"], 20, origin="centroid")   # b. 旋转 footprint
            # r["geom"] = aff.scale(r["geom"], 0.7, 0.7, origin="centroid"); r["h"] /= 0.7  # c. 缩面积、补高度(GFA 守恒)
    return out
```

要拆楼、细分、按距离收拢这种更复杂的,照抄 `engine/operators.py` 里最像的那个算子改
(`split_to_towers` 拆、`infill` 细分、`concentrate` 按距离、`slim` 塔化)。

---

## 2. 登记:让配方引擎认识它

**不改任何文件**,在 notebook 里运行时登记一行:

```python
import operators as ops
ops.register("my_op", my_op)      # 现在 "my_op" 和内置 9 个算子完全平权
```

> 想永久内置?把函数贴进 `engine/operators.py`,并在文件末尾的 `OPS = {...}` 里加一行
> `"my_op": my_op,`。这样 `regimes.yaml` 随时能用,不必每次在 notebook 登记。

---

## 3. 用进配方:写进 `regimes.yaml` 的 steps

算子登记后,就能像内置算子一样写进某个体制的配方(按序施加):

```yaml
my_regime:
  label: 我的权力体制
  desc: 先锁政府,再用我的算子动居民/开发商,最后加密。
  steps:
    - {op: freeze, who: [state]}
    - {op: my_op, target: [resident, developer], amount: 1.8}
    - {op: densify, target: [resident, developer], far_gain: 1.4}
```

`steps` 里每个 `op` 的名字 = 你登记的名字;其余键 = 传给函数的参数(对上函数签名即可)。
跑进阶 notebook 的「四种权力 → 四种形态」那段,把 `my_regime` 也加进去比较，就看到你的算子长出的形态。

## 4. 自检清单(常见坑)

- [ ] 返回了**新列表**,没有原地改 `recs`?(否则前一个算子的结果被污染)
- [ ] 只动了 `target` 内、`not frozen` 的楼?
- [ ] 改几何后 `geom` 仍是合法多边形?(缩太狠会变空;`aff.scale` 比 `buffer` 安全)
- [ ] 想 GFA 守恒,缩面积时补了高度?想加量,只抬了高度?
- [ ] 单位是**米**(EPSG:32651),不是经纬度。

> 诚实边界:你造的 regime→形态 是**教学假设 / 可争论的语言**,不是规划预测。算子不含退线/日照/产权/参与。
> 这正是重点——把「某种权力如何重写城市」写成**可读、可改、可反驳**的代码。
