# 01_py基礎教學 — 觀念簡報大綱

把 12 本 notebook 裡「真正需要靠比喻說清楚」的觀念抽出來，做成 HTML 簡報的內容骨架。

## 選取原則

不是把每個 API 都搬上投影片，而是只挑三類「值得用比喻講」的點：

1. 心智模型（mental model）：背後的運作直覺，講錯就學歪（例：變數是「名稱綁值」、向量化思維）。
2. 常見誤解（misconception）：新手最容易踩、需要被特別澄清（例：return vs print、編碼不是加密、經緯度不能直接算面積）。
3. 跨章重複出現的大觀念：在多本反覆用到，值得獨立成「核心認知」單元（例：可變 vs 不可變、布林遮罩、序列化）。

每個觀念在簡報裡用固定四格呈現：**白話定義 → 比喻 → 心智模型（一句話 / 一張示意）→ 常見誤解**。

## 第二部分 · 各章必提觀念

每章列「需要特別說明的觀念」+ 一句為什麼 + 比喻方向。括號標 [核心Cx] 表示回扣上面的跨章認知。

### PY01 核心語法
- 變數 = 名稱綁定值，型別由值自動決定 [核心C1]：地基觀念。比喻：行李吊牌。
- 真值測試 truthiness [核心C3]：條件不必寫成 `== True`。比喻：空籃子就是沒有。
- 累加器 accumulator（先設 0 再逐筆加）：計次與求和的通用套路。比喻：計分板 / 撲滿。
- while 二分逼近收斂：次數不固定、逼到夠準為止。比喻：猜數字一直對半縮範圍。
- f-string 格式規格（`:.4f`、`:,`）：輸出是最後一哩路。比喻：報表模板填空、對齊。

### PY02 資料結構與推導式
- 四容器各自定位 list / dict / tuple / set：選錯容器後面全卡。比喻：排隊隊伍 / 電話簿 / 刻好的銘牌 / 只收不重複的集郵冊。
- 可變 vs 不可變 [核心C2]：tuple 改不得、list 隨時改。比喻：白板 vs 石碑。
- 推導式（一行做「轉換 + 過濾」）：Python 風格代表。比喻：生產線 + 濾網。
- 產生器與惰性求值（用完即空）[核心C6]：省記憶體但只能走一次。比喻：號碼牌抽一次就沒了。
- enumerate / zip + 拆包：迴圈寫得乾淨。比喻：zip 像兩條拉鍊對齊咬合。

### PY03 函式與程式組織
- 函式 = 封裝可命名、可重用的邏輯；return vs print [常見誤解]：return 把成品交回、print 只是展示。比喻：工廠出貨 vs 公告欄貼出來看。
- 參數種類 + `*args` / `**kwargs`：呼叫彈性。比喻：餐廳固定餐 / 客製 / 任意加點。
- 純函式 vs 副作用：好測試好重用。比喻：自動販賣機（同投幣同出貨）vs 會偷改隔壁的人。
- 作用域 / 閉包 / nonlocal [核心C1]：變數由內往外找；閉包記住外層變數。比喻：俄羅斯娃娃由內往外找；隨身帶小抄。
- 遞迴 + 終止條件 base case：一定要有最小可解的一步。比喻：鏡子照鏡子，總得有盡頭。
- 型別提示（只註記、不強制）：比喻：包裝上的標示，說明但不阻止你。

### PY04 NumPy 數值運算
- array（同型別 + 有形狀）vs list：為什麼換掉 list。比喻：整齊格子盒 vs 雜物抽屜。
- 向量化思維 [核心C4]：先想能不能整批算。比喻：工廠整批 vs 手工逐個。
- 廣播 broadcasting（形狀自動對齊）：免迴圈逐元素運算的核心。比喻：一張公告自動貼到每間教室。
- 布林遮罩 boolean mask [核心C12]：用條件一次取 / 改值。比喻：門禁名單只放行 True。
- 軸 axis（axis=0 逐欄 / axis=1 逐列）[常見誤解]：方向最容易記反。比喻：表格往下加總 vs 往右加總。
- 種子 seeding（可重現）：比喻：同一副洗好的牌、每次從同起點發。

### PY05 pandas 表格資料
- DataFrame = 有標籤的二維表（多個 Series 並排）：一切操作靠標籤定位。比喻：Excel 表 + 每列每欄都有名牌。
- 布林遮罩過濾列 [核心C12]：延續 NumPy 的同一招。比喻：篩子只留符合條件的列。
- groupby 分組聚合：把細項壓成摘要。比喻：學生按班級分堆，再各算平均。
- 缺失值 NaN [核心C12]：會汙染統計，要先決定丟或補。比喻：問卷漏填的空格。
- 降階再升階（to_numpy ↔ DataFrame）[核心C4]：算數值時脫名牌、算完戴回。比喻：比賽脫外套、賽後再穿上。

### PY06 matplotlib 製圖（基礎）
- 狀態式 vs 物件式（fig / ax）[核心C10]：figure 是整塊畫布、axes 是其中一個座標系。比喻：對「當前畫布」喊話 vs 指名某張子圖作畫。
- figsize × dpi = 畫素；批次作圖要 close：比喻：沖洗照片＝尺寸 × 解析度；畫完的畫布要收走別堆積。
- 選圖原則（資料形態配圖型）：折線看趨勢 / 散點看分布 / 長條比類別 / 直方看分布。比喻：看料理選餐具。
- `set_aspect('equal')` 不變形：比喻：地圖 x、y 比例一致，圓才不會被壓扁。
- 標註讓圖能被獨立讀懂：比喻：展品旁的說明牌。
- 中文字型（豆腐字成因）：比喻：沒裝對字型像缺字的印章蓋出空格。

### PY07 matplotlib 製圖（進階）
- 顏色的本質：數值先正規化到 0~1 再查 colormap [核心C9 同源的「換算才有意義」]：比喻：先把分數換成百分比，再對照色卡。
- 多圖比較必須共用色階：比喻：多張地圖要用同一張圖例才能互比。
- 多子圖 + 迴圈批次填圖 [核心C10]：比喻：相框牆逐格放照片。
- 動畫 = 逐格更新同一張圖（frame + update 函式）：比喻：翻頁動畫書。
- 3D 視角 view_init（仰角 / 方位角）：比喻：繞著模型走動看不同角度。
- 圖內嵌：存進記憶體 → base64 → 嵌 HTML [核心C8]：比喻：把照片直接縫進信裡，而非另附附件。

### PY08 物件導向 OOP
- class = 資料藍圖、instance = 照藍圖造出的物件：最關鍵的一組對照。比喻：餅乾模型 vs 烤出的餅乾。
- 封裝：把資料與行為綁在一起（為什麼從 dict 進化到 class）[核心C2 資料+行為]：比喻：瑞士刀（工具與功能合一）vs 散落的工具。
- `__init__` 與 self（self = 物件自己）：比喻：表格上的「本人」欄位。
- 方法鏈 return self：比喻：一條龍服務，一個接一個動作。
- 繼承 inheritance + `super()`：先承父業、再加自己的。比喻：接手家業先繼承既有的，再添新東西。
- 方法覆寫 + 多型（共同介面，如 forward）[核心C5]：比喻：同一顆「發聲」按鈕，各動物叫聲不同。
- dict 還是 class（避免過度設計）：比喻：便條紙 vs 正式表單，看場合用。

### PY09 例外與檔案 I/O
- 例外處理 try / except / else / finally + raise [核心C12]：出錯也要可控。比喻：安全網；finally 是「不論如何都收尾」（關門關燈）。
- with 上下文管理器（保證釋放）[核心C7]：比喻：自動關門的旋轉門。
- 編碼 encoding = utf-8（中文亂碼根源）[核心C8]：比喻：兩人要用同一套密碼本才看得懂。
- pathlib 路徑物件（用 `/` 組合）：比喻：用樂高拼路徑，比手寫地址不易出錯。
- 序列化 serialization（物件 ↔ 文字 / 位元組）[核心C8]：比喻：家具拆平裝箱、到了再組回。
- pickle 僅限信任來源、base64 非加密 [常見誤解 / 安全]：比喻：別人寄來的「組裝說明」可能藏陷阱。

### PY10 命令列 CLI
- 命令列引數：靠外部輸入改行為、不必改原始碼 [核心C10]：比喻：同一台咖啡機按不同按鈕。
- sys.argv 的限制（`[0]` 是腳本名、全是字串、無驗證）：比喻：最原始的紙條傳話，全靠自己看懂。
- argparse（自動解析 + 型別 + 預設 + help）：比喻：正式報名表單，有欄位驗證與說明。
- `if __name__ == "__main__"` [常見誤解]：區分「親自執行」與「被匯入」。比喻：店「自己營業」vs「設備被別店借用」，主流程只在親自開門時跑。
- subprocess + 回傳碼 return code：比喻：派人去辦事，看回報的成功 / 失敗碼。
- exit code 可組合 composable：比喻：接力棒順利交接下一棒。

### PY11 地理空間與圖論
- 向量幾何 = 空間問題的最小積木（Point / Polygon）：比喻：用樂高的點線面拼出形狀。
- CRS 座標系與單位（度 vs 公尺、何時轉換）[核心C9、常見誤解]：經緯度不能直接算面積。比喻：量土地要用公尺的捲尺，不是經緯度的格子。
- 空間索引 STRtree 粗篩（避免 O(n²)）[核心C11]：比喻：先用外框快速排除，再細看誰真的相交。
- GeoDataFrame = 表格 + geometry 欄 + CRS：比喻：名冊多加一欄「位置」。
- OSM 標籤 tag + 工程意識（連網 / 授權 / 離線備援 / 缺值）：比喻：眾人協作的公共地圖，標籤常缺漏要先想備案。
- KD-tree 最近鄰 [核心C11]：比喻：用分區地圖快速找最近的店家。
- 圖 node / edge + 中心性 / 社群：誰是橋樑（介數）、哪群是小圈圈（社群）。比喻：人脈網。
- 向量 vs 柵格 raster：比喻：描邊地圖 vs 馬賽克像素圖。

### PY12 Web / API / 文件工具
- HTTP 三步：送請求 → 看狀態碼 → 取資料：比喻：點餐 → 看收據成不成立 → 取餐。
- 狀態碼 2xx / 4xx / 5xx [常見誤解]：先確認成功再解析 json。比喻：回條上的綠燈 / 紅燈。
- REST API 端點 + 金鑰（API key 不可硬編 / 外洩）：比喻：服務窗口 + 識別證，識別證等於密碼別外流。
- base64 編碼 ≠ 加密 [核心C8、與 PY09 回扣]：比喻：換句話說，不是上鎖。
- URL 解析 + MIME 型別：比喻：把地址拆成各部位；副檔名像包裹上標示內容物。
- 資料與模板分離 + 批次 / 容錯 / 冪等 idempotency [核心C10、C12]：比喻：公版表格填不同資料；流水線壞一筆跳過不停線、重跑結果一致。

## 第三部分 · 簡報產出架構建議

- 形式：HTML 簡報（建議 reveal.js，CDN 載入、單檔可離線開）。
- 拆法：1 份「核心認知總覽」+ 12 份各章簡報（PY01–PY12），或合成一份大簡報用分節。
- 每個觀念固定版型（一張或兩張投影片）：
  1. 觀念名（中文 + 英文原詞）
  2. 一句白話定義
  3. 比喻（一張示意 / 一句類比）
  4. 心智模型重點（2–3 條）
  5. 常見誤解（若有，紅字標出）
- 風格：純文字、無 emoji；術語首見附英文；程式片段只示意語法形狀，不放整段可執行邏輯（與 notebook 分工：簡報講「觀念與比喻」，notebook 講「怎麼做」）。
- 每章開頭放該章「核心認知回扣」徽章（標示用到哪些 C1–C12），讓跨章觀念串起來。
