# 教學 Notebook 撰寫原則

所有 `NN_*.ipynb` 共用。分兩部分：
- **第一部分**：Markdown cell 原則（講概念）
- **第二部分**：Code block 註解原則（講程式）

通則：
- 每本 notebook 完全獨立自足，不引用任何外部專案資料夾或檔名。
- 不使用任何 icon / emoji，純文字。
- 繁體中文為主，術語首次出現附英文原詞（例如「廣播 broadcasting」）。
- 程式變數 / 函式名維持英文，不翻譯。

# 第一部分 · Markdown Cell 原則

## M1. 職責分工
- Markdown 講「**為什麼 / 是什麼 / 何時用**」。
- Code 講「**怎麼做**」。
- Markdown 內**不放可執行邏輯**；要示意語法形狀時，用不執行的 code 區塊展示。

## M2. 標題層級
- `#`（H1）：整本 notebook 只用一次，當檔案大標題。
- `##`（H2）：每個大節（一個概念群）。
- `###`（H3）：節內子項。
- 不跳級（H2 後不直接跳 H4）。

## M3. 每本固定骨架
| 順序 | Cell | 內容 |
|---|---|---|
| 1 | Markdown | H1 標題 + 一句本書定位 + 「學習目標」3-5 條 |
| 2 | Code | 環境 / 字型 setup（會畫圖才放） |
| 3… | Markdown→Code 交替 | 每概念：先 Markdown 講解 → 再 Code 範例 |
| 倒數 2 | Code | 「練習」TODO |
| 末 | Markdown | 「小結」（不放跨筆記本的銜接段落） |

## M4. 概念講解 Cell 寫法（順序）
1. 一句**白話定義**（這是什麼）。
2. **為什麼要學 / 何時用**（動機）。
3. 必要時用**表格**整理（多選項、參數、對照）。
4. 可附**語法形狀**示意（不執行的 code 區塊），例如：
   ```
   [運算式 for 變數 in 可迭代物 if 條件]
   ```
5. 把實際可跑範例**留給下一個 code cell**。

## M5. 長度與聚焦
- 一個 Markdown cell 聚焦**一個概念**，約 3-8 行。
- 概念多就拆多個 cell，搭配各自的 code cell，不要一坨長文。

## M6. 用語規範
- 固定段名：「學習目標」「練習」「小結」（不加符號 / emoji）。
- 術語首見「中文（English）」格式；之後用中文即可。
- 避免口語贅詞；直述。

## M7. 表格使用時機
- 多個並列選項 / 參數 / 方法對照時用表格。
- 流程步驟用有序清單（1. 2. 3.）。
- 重點提醒用無序清單。

## M8. Markdown 範例（好 vs 不好）
好：
```markdown
## 串列推導式 list comprehension

用一行從舊 list 產生新 list，取代 for 迴圈 + append。
資料量大、邏輯單純時更短更快。

形狀：[運算式 for 變數 in 可迭代物 if 條件]
```
不好（混入可執行邏輯、無動機、過長）：
```markdown
## comprehension
xs=[i for i in range(100)] 然後再 ys=[...] 接著 zs=... （一大段塞滿）
```

# 第二部分 · Code Block 註解原則

## C1. 標記系統（純文字，固定五種）
| 標記 | 用途 | 位置 |
|---|---|---|
| `# 概念：xxx` | 區塊標題，標示一個新教學點 | 該段程式**上方**獨立一行 |
| `# 說明文字` | 解釋這一行做什麼 | 該行程式**右側**（行內） |
| `# 注意：xxx` | 常見坑 / 易錯 / 報錯 | 相關行上方或右側 |
| `# 技巧：xxx` | 延伸技巧 / 更好寫法 | 相關行下方或右側 |
| `# TODO:` | 練習留白，學生自己填 | 練習 cell |

## C2. 註解密度（核心原則）
- 概念**首次出現**：必註（`# 概念：`）。
- 同概念**重複使用**：不再註（避免雜訊）。
- **顯而易見**的行（`print(x)`、`import os`）：不註。
- 寧可 Markdown 多講，行內註解精準少量。

## C3. 註解講「為什麼」，不複述程式
不好（複述程式碼，零資訊）：
```python
x = x + 1   # x 加 1
```
好（講意圖 / 原因）：
```python
x = x + 1   # 補上被 0 索引少算的 1 格
```

## C4. 行內註解格式
- 程式與 `#` 之間**至少兩個空格**。
- 同一區塊的行內註解**盡量對齊**，易讀。
```python
x = np.linspace(0, 10, 100)   # 0 到 10 取 100 個等距點
y = np.sin(x)                 # 對每點取 sin（向量化，無需迴圈）
```

## C5. 一個 cell 一個概念
- 一個 code cell 教一個概念；牽涉前置就用前一個 cell 鋪陳。
- cell 第一行常是 `# 概念：...` 點題。

## C6. 好名字 > 註解
- 變數命名清楚就少註解：用 `count`、`heights` 而非 `c`、`h`。
- 例外可用單字母：迴圈 `i, j`；數學慣例 `x, y, z`。

## C7. 範例自足
- 範例資料**自己造**（numpy / list 生成假資料），不讀外部檔。
- 造假資料的行加註說明，例如：
```python
# 造一組假身高資料（公分）當示範用
heights = np.array([158, 172, 165, 180, 169])
```
- 每個範例 cell 都要有**可見輸出**（`print` 或畫圖）。

## C8. 練習（TODO）格式 — 三難度層 + 複合但快速
每本 notebook 固定 **3 題**，一題一個難度層，全部是**複合型**（整合多概念）但**簡短可快速完成**。
不要出太多、太難、太瑣碎的小題。每題都自成一個小任務。

三難度層定義：
| 層級 | 對象 | 設計準則 |
|---|---|---|
| **簡單（新手）** | 完全新手 | 整合 2 個概念；步驟少、路徑直接；幾行就能完成，建立信心 |
| **中間（複合但不難）** | 稍有基礎 | 整合 3-4 概念；複合型但流程清楚，照步驟即可，無需巧思 |
| **稍難（需要思考）** | 有基礎的老手 | 整合多概念 + 一點設計 / 演算法思考（條件組合、收斂、聚合）；仍簡短 |

題目主題原則：
- **貼近真實專案情境**（都市建築量體：樓高、面積、樓地板、容積率、建蔽率；分類標籤；分群；網格聚合；情境前後比較等），讓學生學到能直接遷移的技能。
- 但**仍完全自足**：資料自己用 numpy / list 造（仿真即可），**不引用任何外部專案資料夾或檔名**。

每題提供：**情境** + **要求（分步驟）** + **驗收標準**；留 TODO 骨架不附解答。

格式範例（一題的骨架）：
```python
# TODO 1 ·（簡單）標題（整合：概念A + 概念B）
#   情境：用一句話描述要解決什麼（貼近建築 / 都市資料）。
#   要求：
#     1. 步驟一（用到概念A）
#     2. 步驟二（用到概念B，印結果 / 畫圖）
#   驗收：應該看到 ____（明確輸出或圖形）
# TODO: 在這裡完成
```
- 標題開頭標 `（簡單）/（中間）/（稍難）`，括號內列整合了哪些概念。

## C9. 禁止事項
- 不寫 `# 對應原始碼：...` 之類引用外部專案的註解。
- 不放 emoji / icon。
- 不寫與程式無關的閒聊註解。
